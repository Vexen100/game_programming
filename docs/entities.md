# Игровые сущности

## Назначение документа

Этот документ описывает основные игровые сущности первого прототипа.

Сущности должны быть максимально простыми. На первом этапе важна не визуальная часть, а поведение объектов и их взаимодействие друг с другом.

---

# Player

## Описание

Игрок — управляемый персонаж, представленный простым квадратом.

Игрок должен уметь:
- двигаться по карте;
- сталкиваться со стенами;
- атаковать врагов;
- получать урон;
- умирать и перезапускаться.

## Основные поля

- `position`
- `velocity`
- `rect`
- `health`
- `max_health`
- `damage`
- `speed`
- `attack_range`
- `attack_cooldown`
- `attack_timer`
- `is_alive`

## Основные методы

- `handle_input()`
- `move()`
- `attack()`
- `take_damage(amount)`
- `die()`
- `update()`
- `draw()`

## Состояния

- `idle`
- `moving`
- `attacking`
- `dead`

---

# Enemy

## Описание

Враг — простой противник, представленный квадратом.

На первом этапе нужен только один тип врага.

Он должен:
- находиться на карте;
- замечать игрока в радиусе;
- преследовать игрока;
- атаковать игрока;
- умирать при потере здоровья.

## Основные поля

- `position`
- `velocity`
- `rect`
- `health`
- `max_health`
- `damage`
- `speed`
- `detection_radius`
- `attack_range`
- `attack_cooldown`
- `attack_timer`
- `state`
- `is_alive`

## Основные методы

- `patrol()`
- `chase_player(player)`
- `attack_player(player)`
- `take_damage(amount)`
- `die()`
- `update(player)`
- `draw()`

## Состояния

- `idle`
- `patrol`
- `chase`
- `attack`
- `dead`

## Минимальная логика поведения

- если игрок мертв — ничего не делать;
- если игрок в радиусе атаки — атаковать игрока;
- иначе если игрок в радиусе обнаружения — двигаться к игроку;
- иначе — стоять или патрулировать.

# CapturePoint

## Описание

Точка захвата — объект, который определяет контроль над частью территории.

На первом этапе нужна одна точка захвата.

Она должна:
- иметь владельца;
- иметь радиус захвата;
- накапливать прогресс захвата;
- менять владельца при достижении нужного прогресса.

## Основные поля

- `position`
- `radius`
- `owner`
- `capture_progress`
- `capture_speed`
- `decay_speed`
- `is_contested`

## Возможные значения `owner`

- `enemy`
- `neutral`
- `player`

## Основные методы

- `update_capture(player, enemies)`
- `change_owner(new_owner)`
- `is_player_near(player)`
- `are_enemies_near(enemies)`
- `draw()`

## Минимальная логика захвата

- если игрок рядом и врагов рядом нет — увеличивать прогресс захвата;
- если враги рядом — останавливать захват или уменьшать прогресс;
- если прогресс достиг максимума — владелец точки становится `player`.

# Region

## Описание

Регион — логическая часть мира, которая может принадлежать игроку или врагу.

В первом прототипе может существовать только один регион.

## Основные поля

- `name`
- `owner`
- `player_influence`
- `enemy_influence`
- `capture_points`
- `enemies`

## Возможные значения `owner`

- `enemy`
- `contested`
- `player`

## Основные методы

- `update_influence()`
- `update_owner()`
- `draw_debug_info()`

## Минимальная логика региона

- если точка захвата принадлежит игроку — влияние игрока растет;
- если рядом есть враги — влияние врага растет;
- если влияние игрока выше нужного значения — регион считается захваченным.

---

# Attack

## Описание

Атака может быть отдельной сущностью или простой проверкой радиуса.

Для первого прототипа лучше не создавать отдельный объект атаки, а сделать простую проверку расстояния между игроком и врагом.

## Минимальная логика

- если игрок нажал кнопку атаки;
- и враг находится в радиусе атаки;
- и атака не на перезарядке;
- тогда враг получает урон.

# Общие принципы

## На первом этапе

Все сущности должны быть максимально простыми.

Разрешено:
- использовать прямоугольники вместо спрайтов;
- хранить значения прямо в объектах;
- не делать сложную иерархию классов;
- не добавлять лишние абстракции.

## На следующих этапах

После появления рабочего ядра можно добавить:
- разные типы врагов;
- разные типы точек захвата;
- здания;
- ресурсы;
- квестовых NPC;
- улучшения героя;
- карту мира.
